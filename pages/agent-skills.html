CLI / AGENT SKILL

文字起こしを、次の作業へ。 Take your transcript into the next task.

CLIはkAIgiの録音と文字起こしを操作するプログラムです。Agent Skillは、CodexやClaude CodeがそのCLIを安全に使うための説明書です。 The CLI is the program that operates on kAIgi recordings and transcripts. The Agent Skill is the instruction manual that teaches Codex and Claude Code how to use that CLI safely.

kAIgi 0.2.10

CLIとSkillsの違い CLI and Skills are different

CLI：実際に動く道具 CLI: the tool that runs

ターミナルから実行するkAIgiのプログラムです。録音を探す、文字起こしを読む、音声ファイルを文字起こしする、といった処理を行います。 A kAIgi program run from Terminal. It finds recordings, reads transcripts, and transcribes audio files.

Skill：使い方を教える説明書 Skill: the instruction manual

CodexやClaude Codeに、どのコマンドをどの順番で使うか、文字起こしをどう確認・修正するかを伝えます。Skillだけでは録音を操作できません。 It tells Codex and Claude Code which commands to use, in what order, and how to review and correct transcripts. The Skill cannot operate recordings by itself.

設定では、まずCLIとkAIgi Skillをそれぞれインストールします。その後、CodexやClaude CodeでSkillを呼び出すと、CLIを使った手順を案内します。 In Settings, install the CLI and kAIgi Skill separately. Then invoke the Skill in Codex or Claude Code; it guides the CLI workflow.

Invoke

Skillsの呼び出し方 How to invoke the Skill

Codex

自然文で指定 Name it in your request 
kAIgi skillを使って、選択した文字起こしを2段階で高精度化して

CodexのSkills一覧から kaigi を選べる場合は、そこから指定します。環境によっては $kaigi と表示されます。 If your Codex surface has a Skills picker, select kaigi. Some surfaces show the explicit mention as $kaigi.

Claude Code

スラッシュコマンド Slash command 
/kaigi 選択した文字起こしを2段階で高精度化して

入力欄で / を入力すると、利用できるSkillsを一覧できます。依頼内容に応じて自動で読み込まれることもあります。 Type / to see available Skills. Claude Code may also load it automatically when the request matches its description.

Commands

コマンド一覧 Command list

録音IDは最初に list または search で確認します。 Find a recording ID with list or search first.

kaigi recordings list --limit 20

録音を探す Find recordings

録音タイトル、日時、長さ、音声・文字起こしの有無を一覧します。 Lists titles, dates, durations, and whether audio and transcripts exist.

kaigi recordings inspect <recording-id> --max-segments 500

全文を確認 Inspect a transcript

話者・時刻・確度と一緒に文字起こしを確認します。 Inspects the transcript with speakers, timing, and confidence.

kaigi recordings search "query" --limit 20

録音内を検索 Search transcripts

録音タイトルと文字起こし本文から録音を絞り込みます。 Searches recording titles and transcript text.

kaigi recordings export <recording-id> --out "$working"

編集用に取り出す Create a working copy

録音IDと元データのハッシュを含む編集用コピーを作ります。正式データは直接編集しません。 Creates a recording-bound working copy. Do not edit canonical files directly.

kaigi recordings apply-transcript <recording-id> --in "$working"

修正を反映 Apply corrections

検証した文字だけの修正をkAIgiへ反映します。古いコピーは拒否されます。 Validates and applies text-only corrections. Stale working copies are rejected.

kaigi transcribe /path/to/audio.m4a --quality accurate --out /tmp/kaigi-transcription.json

音声ファイルを文字起こし Transcribe an audio file

外部音声をMac上で文字起こしします。結果はJSONで、録音一覧には自動追加しません。 Transcribes an external audio file on this Mac. The JSON result is not added to the library automatically.

Two-pass refinement

高精度化は2段階 Refine in two passes

1. 確度の低い箇所を直す 1. Fix low-confidence segments

needsReview または確度 0.70 未満を、前後の発話や固有名詞と照合します。確信できない部分は残します。 Review needsReview segments or confidence below 0.70 using nearby speech and names. Leave uncertain wording unchanged.

2. 全体の文脈で整える 2. Normalize with full context

全文を読み直し、名前・製品名・専門用語・句読点を一貫させます。要約や発話の追加はしません。 Re-read the full transcript and normalize names, products, technical terms, and punctuation. Never summarize or add speech.

3. 最後に一度だけ反映 3. Apply once at the end

2段階が終わってから apply-transcript を実行し、kAIgi上で結果を確認します。 Run apply-transcript only after both passes, then verify the result in kAIgi.

プライバシーの確認 Privacy check

CLIはこのMac上で処理します。kAIgiアプリはCLIを初めてインストールする際、CodexやClaude Codeのモデルによっては文字起こしがモデル提供元へ送られる可能性があることを警告します。0.2.10ではコマンドごとの確認は行いません。 The CLI processes data on this Mac. kAIgi shows a warning when you install the CLI for the first time because the selected Codex or Claude Code model may receive transcript text. kAIgi 0.2.10 does not ask for confirmation before every command.

list はタイトルや日時などのメタデータ、inspect・search・export・apply-transcript は文字起こし本文を扱います。AI ResearchのOpenAI API連携は別の機能です。詳しくはプライバシーポリシーをご確認ください。 list handles metadata such as title and date; inspect, search, export, and apply-transcript handle transcript text. AI Research’s OpenAI API integration is separate. See the Privacy Policy.
