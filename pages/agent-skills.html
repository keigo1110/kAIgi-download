CLI / SKILLS

文字起こしを、次の作業へ。 Take your transcript into the next task.

kAIgi CLIは録音と文字起こしをローカルで扱い、Agent SkillはCodexやClaude Codeに安全な使い方を伝えます。低確度の箇所を直し、全体の文脈で整えた結果をkAIgiへ戻せます。 The kAIgi CLI works with recordings and transcripts locally. The Agent Skill teaches Codex and Claude Code how to use it safely, refine low-confidence text, improve the full context, and return the result to kAIgi.

kAIgi 0.2.11 kAIgi 0.2.11

まず設定でインストール Install from Settings first

1. CLIをインストール 1. Install the CLI

kAIgiの設定にある「Codex / Claude Code連携」でCLIをインストールします。保存先も同じ画面で確認できます。 In kAIgi Settings, open “Codex / Claude Code Integration” and install the CLI. The destination is shown there.

2. Skillsをインストール 2. Install the Skills

同じ画面からCodexとClaude Code向けのkAIgi Skillをインストールします。保存先はFinderで開けます。 Install the Skill for Codex and Claude Code from the same screen. Each destination can be opened in Finder.

3. Skillsを呼び出す 3. Invoke the Skill

CodexではSkill名を指定し、Claude Codeでは /kaigi を入力します。 Name the Skill in Codex; type /kaigi in Claude Code.

Commands

コマンド一覧 Command list

録音IDは最初の list または search で確認します。パスに空白がある場合は引用符で囲んでください。 Find a recording ID with list or search first. Quote paths that contain spaces.

kaigi recordings list --limit 20 --best-effort

録音を探す Find recordings

録音タイトル、日時、長さ、音声・文字起こしの有無を一覧します。古い録音の一部が読めない場合も、ロック確認を保ったまま理由と件数を返します。 Lists recording titles, dates, durations, and whether audio and transcripts exist. If legacy sessions cannot be read, it keeps the lock checks and reports reason/count metadata.

kaigi recordings inspect <recording-id> --max-segments 500

全文を確認 Inspect a transcript

選んだ録音の文字起こしを、話者・時刻・確度と一緒に確認します。 Inspects the selected transcript with speakers, timing, and confidence.

kaigi recordings search "query" --limit 20 --best-effort

録音内を検索 Search transcripts

録音のタイトルと文字起こしから、該当する録音を絞り込みます。--best-effort は読めない録音を止めずに件数で知らせます。 Searches recording titles and transcript text. --best-effort reports inaccessible recordings by count instead of aborting discovery.

kaigi recordings export <recording-id> --out "$working"

編集用に取り出す Create a working copy

録音IDと元データのハッシュを含む、録音に紐づいた編集用コピーを作ります。直接ファイルを編集しません。 Creates a recording-bound working copy with the recording ID and source hash. Do not edit the canonical files directly.

kaigi recordings validate-transcript <recording-id> --in "$working"

反映前に検証 Preflight corrections

録音が変わっていないか、文字だけの修正か、何セグメント変わるかを確認します。kAIgiの正式データは変更しません。 Checks the source hash, text-only contract, and exact changed-segment count without modifying kAIgi.

kaigi recordings apply-transcript <recording-id> --in "$working"

修正をkAIgiへ反映 Apply corrections

文字だけの修正を検証し、kAIgiの正式データへ安全に反映します。別の変更が先に入っていれば、古いコピーとして拒否します。 Validates text-only corrections and safely applies them to kAIgi. A stale working copy is rejected if another change landed first.

kaigi recordings discussion-log export <recording-id> --out "$context"

議論ログの材料を取り出す Export discussion-log context

全文、録音ID、文字起こしハッシュ、発話列、現在の議論ログを、エージェントが後処理できる作業用JSONとして取り出します。元の録音データは変更しません。 Exports the full transcript, recording ID, source hash, utterance list, and current discussion log as a working JSON for an agent. It does not change the recording.

kaigi recordings discussion-log validate <recording-id> --in "$context"

議論ログを反映前に検証 Validate a discussion log

discussion_logのスキーマ、発話の完全一致、セグメントの連続被覆、議題ブロック、録音ハッシュを確認します。正式データは変更しません。 Checks the discussion_log schema, exact utterance identity, contiguous segment coverage, topic blocks, and recording hash without changing canonical data.

kaigi recordings discussion-log apply <recording-id> --in "$context"

議論ログを反映 Apply a discussion log

検証済みの議論ログだけを、セッションの排他的ロック下で discussion_log.json へ原子的に反映します。文字起こしが先に変わっていれば古い作業コピーとして拒否します。 Publishes only the validated discussion log to discussion_log.json under the session's exclusive lock. It rejects a stale working copy if the transcript changed first.

kaigi transcribe /path/to/audio.m4a --quality accurate --out /tmp/kaigi-transcription.json

音声ファイルを文字起こし Transcribe an audio file

外部音声ファイルを端末内で文字起こしします。結果は独立したJSONで、録音一覧へは自動追加しません。 Transcribes an external audio file on this Mac. The result is a standalone JSON file and is not added to the recording library automatically.

Two-pass refinement

高精度化は2段階 Refine in two passes

1. 確度の低い箇所から 1. Start with low confidence

needsReview または確度 0.70 未満を、前後の発話・固有名詞・繰り返し表現と照合して直します。確信できない部分は残します。 Review needsReview segments or confidence below 0.70 using nearby speech, names, and repeated terms. Leave uncertain wording unchanged.

2. 全体の文脈で整える 2. Normalize with full context

全文を読み直し、名前・製品名・専門用語・句読点・文の切れ目を、意味が変わらない範囲で一貫させます。要約や発話の追加はしません。 Re-read the full transcript and normalize names, products, technical terms, punctuation, and sentence boundaries without changing meaning. Never summarize or add speech.

3. 検証して一度だけ反映 3. Preflight, then apply once

2段階が終わったら validate-transcript で変更件数を確認し、その後に apply-transcript を一度だけ実行します。反映後は新しいパスへ再度exportし、意図した変更だけか確認します。 After both passes, check the change count with validate-transcript, then run apply-transcript once. Export to a new path afterward and verify that only the intended text changed.

プライバシーの確認 Privacy check

CLIは音声と文字起こしをこのMac上で処理します。kAIgiアプリはCLIを初めてインストールする際、CodexやClaude Codeのモデルによっては文字起こしがモデル提供元へ送られる可能性があることを警告します。0.2.11ではコマンドごとの確認は行いません。list はタイトルや日時などのメタデータ、inspect・search・export・apply-transcript・discussion-log は文字起こし本文を扱います。 The CLI processes audio and transcripts on this Mac. kAIgi shows a warning when you install the CLI for the first time because the selected Codex or Claude Code model may receive transcript text. kAIgi 0.2.11 does not ask for confirmation before every command. list handles metadata such as title and date; inspect, search, export, apply-transcript, and discussion-log handle transcript text.

AI ResearchのOpenAI API連携は別の機能です。詳しくはプライバシーポリシーを確認してください。 AI Research’s OpenAI API integration is a separate feature. See the Privacy Policy for details.
